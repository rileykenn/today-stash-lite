3. Bonus — WebSite Schema (adds a Google Sitelinks Search Box)
Worth adding to the homepage too:
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "Today's Stash",
  "url": "https://www.todaysstash.com.au",
  "potentialAction": {
    "@type": "SearchAction",
    "target": {
      "@type": "EntryPoint",
      "urlTemplate": "https://www.todaysstash.com.au/consumer?q={search_term_string}"
    },
    "query-input": "required name=search_term_string"
  }
}
</script>
(Update the urlTemplate to match however your deal search URL actually works.)

How to implement in Next.js
Since you're on Next.js, the cleanest way is to drop these into your layout or page components using a <Script> tag or directly in <Head>:
// In your _app.js or layout.js
import Head from 'next/head'

export default function Layout({ children }) {
  const orgSchema = {
    "@context": "https://schema.org",
    "@type": "Organization",
    "name": "Today's Stash",
    // ... rest of object
  }

  return (
    <>
      <Head>
        <script
          type="application/ld+json"
          dangerouslySetInnerHTML={{ __html: JSON.stringify(orgSchema) }}
        />
      </Head>
      {children}
    </>
  )
}

Validate before going live
Once you've added these, test them at schema.org/validator or Google's Rich Results Test. Google will show you any errors and confirm what rich result features you're eligible for.
